Domain 6: Secure Software Testing

According to NIST, is a type of assessment method that is characterized by the process of exercising one or more assessment objects under specified conditions to compare actual with expected behavior, the results of which are used to support the determination of security control or privacy control effectiveness over time. Testing software involves the development of both functional and nonfunctional test scenarios and cases.

aims to assess the functionality of the software in respect to functional specifications (i.e., is the code performing the correct actions). aims to assess cyber resiliency, performance, safety, and useability (i.e., do code security and quality align with organizational goals).

Built-in software security capabilities must be functionally tested. To maintain the required levels of confidence, integrity, and availability of software, security practitioners must aid organizations in maturing nonfunctional security testing strategies. is a process and specialization within cybersecurity that seeks to establish the level of confidence that a software is free from vulnerabilities, either intentionally designed into the software or accidentally inserted at any time during its life cycle, and that the software functions as intended by the purchaser or user.

Although software assurance is an integral part of secure software testing, it is important to understand that derivative artifacts from software engineering must also be inspected. For example, software documentation includes installation procedures, error messages, and release notes.

Software defects must be tracked and addressed. Security defects have a life cycle. The Common Vulnerability Scoring System (CVSS) is a published standard used by organizations worldwide to capture the principal characteristics of a vulnerability.

Test data needs to be accurate, precise, and complete. In most cases, using production data in a test environment is not acceptable, especially when addressing data privacy. Software verification and validation (V&V) evaluates how well the software is meeting its technical requirements and its safety, security, and reliability objectives relative to the system.
