Domain Review

Production application security challenges arise when organizational policies and procedures do not include considerations for production aspects of the SDLC. Policies and procedures such as SecCM help ensure production risks are managed consistently and addressed appropriately. Effective and timely response to system security incidents requires planning and preparation.

Post-deployment security checks are important for establishing system confidence and to understand operational security posture. Issues identified in post-deployment testing must be tracked for resolution. Automated security testing facilitates test execution repeatedly, consistently, and rapidly at scale. ISCM provides better risk management decision-making by supporting anomaly detection, incident response, compliance with regulatory requirements, and forensics analysis. Some enterprises are vast, and continuous monitoring can present challenges, though modern automated tools and technologies can facilitate implementation.

In summary, security practitioners must understand that security architecture and secure software development only represent part of the security equation for securing software throughout the life cycle; risk must be managed beyond development by including considerations for secure deployment, operations, and maintenance.
