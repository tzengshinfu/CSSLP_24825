Closing Remarks

Software security practitioners should always consider context and life cycle scope when addressing security challenges. Due care is exhibited when habits, policies, and procedures adequately insulate our organization from risks by incorporating security considerations for both software development operations and secure software development. Due diligence should also be mapped from the beginning of the system life cycle, with the incorporation of functional and nonfunctional security requirements, to the end where software and data are properly decommissioned.

From a simplified point of view, software security must be planned, validated, verified, and documented for every phase in the life cycle. All processes should include the provision to iterate in order to adapt and improve security implementations over time using a data-driven, methodical approach. Always consider confidentiality, integrity, and availability based on organizational needs and when faced with complexity or confusion proactively exercise common sense.
