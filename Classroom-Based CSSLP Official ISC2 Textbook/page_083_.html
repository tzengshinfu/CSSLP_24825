Domain 8: Secure Software Supply Chain

When outsourcing and offshoring arrangements are leveraged for software development, any gains could come at the cost of losing control and visibility into the software development practices and processes. When applications are constructed using many third-party software components (proprietary or open source), vulnerabilities in those components may be inherited by the applications. Supply chain attacks and the risks of the software supply chain must be understood and managed.

Source code repositories represent the crown jewels for most software and technology companies. Safeguards including access controls should be implemented to limit access to reduce the possibilities of attacks such as deletion, exfiltration, and the insertion of malicious logic, which negatively impact source code availability, confidentiality, and integrity, respectively. A source code repository represents valuable intellectual propertyâ€”data at rest. However, the development environment must also be secure, and software must be distributed and transferred securely, which represents protection for valuable data in transit.

Additionally, when acquiring software, organizations must address ownership of intellectual properties, code escrow, liabilities and warranties, and service-level agreements. Supply chain vendors must be categorized and evaluated based on the level of criticality the supplier component brings to business functions. Building a level of confidence in the security of acquired software may require a security audit of the supplier's SDLC, policy, and procedures. Simply stated, security must be addressed in software acquisition processes.

Learning Objectives

In this domain you will:

  • Describe the software supply chain.
  • Review the process of software supply-chain risk management.
  • Explain security risks associated with third-party software.
  • Associate the risks with peer-to-peer applications and file sharing.
  • Interpret code repository and environment security.
  • Express cryptographically hashed, digitally signed components.
  • Identify security requirements and principles in software acquisition.
  • Elaborate critical considerations for software acquisition.
  • Explain contractual requirements for software acquisition.
