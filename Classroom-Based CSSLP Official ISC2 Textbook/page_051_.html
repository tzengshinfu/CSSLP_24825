Domain Review

A plethora of secure coding practices, guidelines, and checklists are in existence today. The same is true of tools and techniques that can be integrated both manually and automatically into the software development life cycle. The software implementation phase represents when software code is actually drafted and eventually stored in various code repositories.

All known software production vulnerabilities that exist today are the result of technical debt, whether intentionally accepted or unintentionally incorporated during development through the introduction of software weaknesses. Secure coding standard, appropriate programming language, organizational administrative control (e.g., SDP, STP), and technical control selection (e.g., SAST, DAST, secure builds) play an important role in software security. Implications for selections continue forward in the life cycle until software is decommissioned. 

To successfully build secure software, organizations must incorporate security practices into the SDLC. Maturity of processes can be measured using maturity models such as CMMI or open-source models such as SAMM or BSIMM. It is easier and more cost-effective to develop organizationally defined, cyber-resilient software during the implementation phase than to attempt to mitigate risk with security controls in a production environment later in the life cycle.
