Discussion: Input Validation

Input validation, also known as input sanitization, provides the first layer of defense for inputs into a software system. Applications are increasingly interdependent (i.e., it takes many applications integrated together to solve a business problem), resulting in many complicated data flows in and between software systems. 

Instructions

As a security practitioner internal to a software producing organization, what activities could help with the identification of these entry points?

  • Although “input validation” and “input sanitization” are often used interchangeably within our industry, are they the same thing? Why?
