Domain 6

Discussion: Severity Scoring

During development at an organization that is not using best practices, software security bugs are detected after the source code is checked into the repository and an issue is added to the backlog to be addressed later. Bug severity likely plays into the prioritization of fixes.

Instructions

Consider the following questions. Be prepared to share your thoughts with the class.

  • What problems could arise from the lack of a standard used to determine the severity of a bug? In other words, how can subjectivity during severity determination of a bug become a problem?
  • What is wrong with this development pipeline?
