Domain 5: Secure Software Implementation

The implementation phase represents the opportunity to build secure, cyber-resilient software by incorporating pre-established architecture requirements and constraints into a tangible system. The earlier security is integrated into software or systems life cycle, the less costly risk management and sustainment activities will be throughout the useful life of the system. Organizations that develop software should establish and enforce secure coding standards that include concepts such as robust input validation, output encoding, strong authentication mechanisms, session management, encryption techniques, and exception management.

Secure coding standards often minimize software defects, also known as weaknesses. Databases such as MITRE Common Weakness Enumeration (CWE) provide a consolidated way to describe common code issues that, when included in source code, are likely to result in software system vulnerabilities later when software is released to production. Additionally, OWASP also releases “Top 10” lists that describe important software security issues that should be addressed during the software development life cycle (SDLC).  

Risks associated with third-party code or libraries must be understood prior to creating dependencies within an organization's source code base. can help organizations gain visibility into open-source inventory and address security as well as license management when software development efforts predate security emphasis.

Note: This domain contains software source code examples. Not every secure software professional will find these examples useful nor is he/she/they expected to be familiar with every programming language or language nuance contained herein, however they are included for learners with software development and assessment backgrounds.

Learning Objectives

In this domain you will:

  • Identify the features of secure coding standards.
  • Describe different approaches for implementing security in managed applications.
  • Identify common flaws in software and corresponding mitigation strategies.
  • Explain common secure coding practices.
  • Identify methods of data protection in transit and at rest.
  • Identify software weaknesses listed in the most common vulnerability lists and databases.
  • Describe the function and methods of software assurance tools.
  • Describe categorization of controls by type and by functionality.
  • Identify controls to prevent common web application vulnerabilities.
  • Describe the process for defining a security strategy that includes considerations for software risks.
  • Identify the risks associated with using third-party and open-source components and libraries.
  • Describe the various integration categories and their relevance to software security.
  • Describe the build automation process.
  • Explain the techniques used in software assurance.
