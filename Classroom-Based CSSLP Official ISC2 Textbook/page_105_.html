Discussion: Security Misconfiguration

The OWASP Top 10 is an awareness document for web application security. It represents the most critical security risks to web applications. One of the Top 10 is “Security Misconfiguration.” This basic understanding will suffice for this activity.

Security misconfiguration can occur at many different system levels within infrastructure, and the results of a single mistake can have devastating organizational consequences. There are many levels of infrastructure required to host an application—application, web server, operating system, network services, database, virtual machine, container, and storage.

Consider the case of enabled directory browsing as an example of security misconfiguration on web servers. Directory browsing allows client machines to request a hierarchical list of contents within the hosted web directory.

Instructions

Be prepared to share your thoughts with the class by addressing the following questions:

  1. What are some of the threats, attack vectors, and potential outcomes that might result when directory browsing is not prevented?
  2. Would you expect that disabling directory browsing per CIS benchmarks (i.e., Level 1) would require a different set of steps for two competing web servers, IIS and Apache? Explain.

Discussion: Comparing Software Security Tools

Your organization is considering licensing one of three static analysis tools to detect defects in source code prior to check-in. The metrics for comparison are as follows:

  1. Number of critical issues found per 100 lines of code (ratio)
  2. Rank of the assurance tools performance (ordinal)
Tool Metric 1 Metric 2 Annual License Cost
A 10 1 $25,000
B 4 2 $1,500
C 3 3 Open Source 

Table 1

The table appears to demonstrate that you get what you pay for. The more the tool costs, the more critical issues were detected within our source code. However, there is a problem with this analysis. You cannot be certain that all results from each tool are valid. There is a false precision problem in our metrics. Suppose the following table is a more accurate table based on our limited precision.

Discussion: Decommissioning Existing Software

Earlier, security misconfiguration was discussed, including the layers of supporting infrastructure beyond the application cited for misconfigurations that result in security issues. The same multilayer infrastructure considerations apply to decommissioning software. If all infrastructure layers that consist of the production software solution are not considered, data remanence and other security issues may occur.  

Instructions

Consider the following question. Be prepared to share your thoughts with the class.

  • Besides the software itself, what else needs to be considered when decommissioning software?
  • How could documentation assist with decommissioning?
