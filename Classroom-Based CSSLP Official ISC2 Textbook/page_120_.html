Question 1

4. Your application needs to bind to a well-known port. For the binding to be successful, elevated privileges are required. The application continues to run with elevated privileges after the binding is completed. This constitutes a violation of which security principle? (D1.2, L1.5)

A. Defense-in-depth
marked correct B. Least privilege
C. Separation of duties
D. Economy of mechanism
Feedback

Your answer is correct.

The correct answer is B. Processes should be granted just enough privileges to be able to complete the task at hand in the shortest time period. Continuing to run with elevated privileges when no longer necessary is a violation of the least privilege principle. The application should downgrade to minimally required privileges when binding has concluded. 

Incorrect answer feedback: Separation of duties relates to the concept that no subject should be given enough privileges to misuse the system on their own. Defense-in-depth involves applying multiple layers of protection, where a subsequent layer will provide protection if a previous layer is breached. Economy of mechanisms demonstrates the principle “Keep it simple, stupid (KISS).”

A. Passwordless user authentication
B. Least privilege
marked correct C. Simplified user authentication
D. Usage of one-time password (OTP) tokens
Feedback

Your answer is correct.

The correct answer is C. Simplification of user authentication is the primary goal for SSO through one set of credentials and reduced sign-ins. It is a good example in the context of the economy of mechanism principle. SSO provides the capability to authenticate once and be automatically authenticated when accessing various additional target systems. SSO is convenient and simplifies the need to remember several passwords.

Incorrect answer feedback: Passwordless user authentication and OTP are distractors. Least privilege refers to properly implementing authorization rather than authentication, which is the main goal of SSO.
