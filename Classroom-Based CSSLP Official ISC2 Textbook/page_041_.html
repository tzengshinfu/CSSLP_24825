Domain Review

Architecting and designing secure systems is not a trivial undertaking. Security practitioners must work with business leaders to justify early investment in the requirements, architecture, and design of secure systems and software. To be considered trustworthy, software systems must undergo an organizationally defined level of rigor to generate compelling evidence to support security claims. Software must meet security requirements to deliver the protection and performance needed to ensure production continuity of security operations. NIST SP 800-160 vol. 1 provides a starting point for ensuring disciplined, structured, and standards-based systems security engineering activities.

Through exposure to the concepts of security architecture, secure system engineering, modularity, interface hardening, threat modeling, and life cycle operations, a blossoming security practitioner begins to understand that application security is more than the sum of the security profiles of each separate component. Security for modern complex software solutions requires a system security engineering mindset and coordination among business, technical, and security stakeholders.
